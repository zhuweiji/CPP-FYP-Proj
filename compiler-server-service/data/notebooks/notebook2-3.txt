# Clean Code

The rationale we have given for the concept of classes and functions in code thus far is to promote re-usability and organization. \n
In fact, this is the primary challenge in software design. The ultimate end goal is to ensure that the number of bugs in our code is minimized while the ease of adding and modifying code is maximised.

We will introduce three different groups of concepts in the next few chapters: \n
1. Relationships
2. Design Principles
3. Design Patterns
